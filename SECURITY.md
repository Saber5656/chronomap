# Security Policy

## Supported versions

Chronomap supports only the latest version deployed from the default branch. Older
deployments, forks, and unreleased local changes are not supported.

## Reporting a vulnerability

Please report suspected vulnerabilities privately through GitHub's
**Security** tab by selecting **Report a vulnerability**. Do not open a public
issue or discussion for a security report.

This project is maintained by one person. Reports are acknowledged on a
best-effort basis, with a target of 14 days, but no response-time guarantee is
offered. Please include reproduction steps, the affected version or commit, the
expected impact, and any suggested mitigation.

## Scope

Chronomap is a client-only web application. Vulnerabilities in Chronomap's code,
build configuration, dependency use, or browser-side security boundaries are in
scope.

Issues that exist only in upstream data or service providers, including the GSI,
Konjaku, and Wikimedia services, are outside this project's scope and should be
reported to the relevant provider. Reports about how Chronomap integrates with an
upstream service remain in scope.
